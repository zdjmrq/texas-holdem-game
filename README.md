## 📖 游戏简介

**德州扑克小游戏** 是一款功能完整的德州扑克桌面应用，基于 **Electron** 构建。它同时支持 **两种游玩模式**：

- 🎮 **本地模式** — 单机与 AI 进行对战，无需联网，开箱即玩
- 🌐 **联网模式** — 通过 WebSocket 连接服务器，与朋友或其他玩家在线对战

游戏支持 **标准德州扑克** 和 **短牌 6+ Hold'em** 两种规则，内置多风格 AI 对手、实时胜率分析、概率计算等丰富功能。

> 绝不作弊！每位 AI 玩家只能看到自己的底牌，通过独立思考做出决策。

## 🚀 快速开始

### 环境要求

- [Node.js](https://nodejs.org/) ≥ 16.x
- npm (随 Node.js 安装)

### 安装客户端依赖

```bash
# 克隆或进入项目目录
git clone https://github.com/zdjmrq/texas-holdem-game.git
cd texas-holdem-game

# 安装依赖
npm install
```

### 运行本地游戏（单机 vs AI）

```bash
# 直接启动 Electron 窗口，无需服务器
npm start
```

游戏即可启动，点击「开始游戏」即可与 AI 对战。

### 运行联网游戏（多人对战）

联网模式需要先启动 WebSocket 服务器，然后通过客户端连接。

## 🖥️ 服务器配置与运行

### 1. 安装服务器依赖

```bash
# 进入 server 目录
cd server

# 安装依赖（仅需 ws 库）
npm install
```

### 2. 启动服务器

```bash
# 默认端口 3000
node index.js

# 指定端口（例如 8080）
node index.js 8080
```

服务器启动后，你会看到：

```
🃏 德州扑克服务器启动 | ws://0.0.0.0:3000
```

### 3. 服务器配置参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `端口` | 命令行参数指定 | `3000` |

示例：

```bash
# 在 5000 端口运行
node index.js 5000
```

### 4. 服务器架构说明

```
server/
├── index.js          # WebSocket 服务器入口，处理连接/消息路由
├── room.js           # 房间管理：创建、加入、离开、玩家状态
├── game-engine.js    # 牌局引擎：发牌、下注轮次、牌型判定、AI决策
├── logger.js         # 全方位运行时日志系统
└── package.json      # 依赖配置（仅 ws）
```

**消息协议：**
- 客户端 ↔ 服务器通过 JSON 消息通信
- 消息类型：`create_room`、`join_room`、`leave_room`、`start_game`、`player_action`
- 服务器广播：`room_created`、`room_joined`、`player_joined`、`player_left`、`game_state`、`game_starting`

### 5. 日志系统

服务器会自动在 `错误日志/logs/` 目录下生成日志文件：

| 日志文件 | 内容 |
|----------|------|
| `action_*.log` | 玩家行动记录 |
| `ai_*.log` | AI 决策过程 |
| `game_*.log` | 游戏事件（开局、摊牌、结束） |
| `state_*.log` | 牌局状态快照 |
| `ws_*.log` | WebSocket 通信日志 |

## 🌐 联网对战使用说明

### 步骤 1：启动服务器

按上方「服务器配置与运行」说明启动服务器。

### 步骤 2：进入游戏

```bash
# 回到项目根目录
cd ..

# 启动客户端
npm start
```

### 步骤 3：连接服务器

1. 在游戏主界面选择 **🌐 联网游戏**
2. 输入服务器地址（默认 `ws://localhost:3000`）
3. 输入你的昵称
4. 点击 **连接**

### 步骤 4：创建或加入房间

- **创建房间**：设置人数、模式（标准/短牌）、AI 难度，点击「创建房间」
- **加入房间**：输入 4 位房间号，点击「加入」

### 步骤 5：开始游戏

- 只有 **房主** 可以开始游戏
- 房间不满员时，剩余位置会自动填充 AI 玩家
- 游戏过程中可随时查看胜率分析

## 🎮 玩法与操作指南

### 游戏流程

1. **发底牌** — 每人获得 2 张仅自己可见的底牌
2. **翻牌前** — 从 UTG 位开始行动，顺时针轮流下注
3. **翻牌** — 发出 3 张公共牌，继续下注
4. **转牌** — 发出第 4 张公共牌，继续下注
5. **河牌** — 发出第 5 张公共牌，最后一轮下注
6. **摊牌** — 剩余玩家亮牌比大小

### 行动选项

| 操作 | 快捷键 | 说明 |
|------|--------|------|
| ❌ 弃牌 | `F` | 放弃本局，失去已投入筹码 |
| ✅ 过牌 | `C` | 无需加注时保持当前注额 |
| 📞 跟注 | `C` | 支付与当前注额相同的筹码 |
| 📈 加注 | `R` | 提高当前注额，拖拽滑块调整金额 |
| 🔴 全下 | `A` | 将所有筹码推入底池 |

### 牌型大小（从大到小）

| 排名 | 牌型 | 标准德州 | 短牌 6+ |
|------|------|----------|---------|
| 1 | 皇家同花顺 | 🏆 | 🏆 |
| 2 | 同花顺 | 🏆 | 🏆 |
| 3 | 四条 | 🏆 | 🏆 |
| 4 | 同花 | 🏆 | ⬆️ 上升 |
| 5 | 葫芦 | 🏆 | ⬇️ 下降 |
| 6 | 顺子 | 🏆 | 🏆 |
| 7 | 三条 | 🏆 | 🏆 |
| 8 | 两对 | 🏆 | 🏆 |
| 9 | 一对 | 🏆 | 🏆 |
| 10 | 高牌 | 🏆 | 🏆 |

> 短牌（6+ Hold'em）规则：同花 > 葫芦，顺子 > 三条。因为去掉了 2-5 的低牌，牌型概率发生变化。

## 🔧 常见问题

**Q: 本地游戏需要启动服务器吗？**
A: 不需要。本地模式完全在 Electron 客户端内运行，开箱即玩。

**Q: 联网模式需要公网 IP 吗？**
A: 局域网内游玩只需内网 IP。公网对战需要端口映射或使用云服务器。

**Q: 最大支持多少人同时游戏？**
A: 每桌 2-10 人，人数不足时自动由 AI 补位。

**Q: 游戏有作弊行为吗？**
A: 没有！AI 玩家只能看到自己的底牌，通过手牌强度和概率计算做出决策，绝不知道其他玩家的牌。
